- **Single Document**: Update specific architecture document
- **Multiple Documents**: Coordinate updates across documentation
- **New Document**: Create using architecture template system

### 3. Execute Updates
- Read current documentation using `mcp-architecture:arch_get_document`

- Update Master Architecture Index if needed

### 4. Validate Integration
- Test navigation pathways from Master Architecture Index

- Confirm documentation reflects actual system behavior
- Store update achievement in brain memory
